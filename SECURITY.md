# Security Policy

This project is a **learning repository** and is not intended for direct
deployment in high-risk production environments without additional hardening.

## Supported Versions

Security-related fixes are made on the main branch as needed. There is no
formal release support policy.

## Reporting a Vulnerability

If you discover a security issue:

1. Do **not** open a public GitHub issue with sensitive details.  
2. Email **scott@johansonllp.com** with:

   - A clear description of the issue  
   - Steps to reproduce  
   - Any suggested mitigations  

We will acknowledge receipt and work to understand and address the issue.
When appropriate, we may document mitigations or learning notes in the repo.
